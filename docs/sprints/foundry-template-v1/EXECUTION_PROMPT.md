### Story 1.2: Create `src/config/foundry-template.ts`

Create this file following the `hero-prompts.ts` pattern. Here is the interface:

```typescript
export interface FoundryTemplate {
  id: string
  title: string
  description: string
  templateText: string
  isPreFilled: boolean
}
```

**CRITICAL:** The `templateText` for the Signal Watch example must include the full 10-section template with all Signal Watch content filled in. This means:

For Section 1 (Domain Problem Statement):
```
## Section 1: Domain Problem Statement [REQUIRED]

Competitive intelligence monitoring for the AI industry. Tracks companies, technologies, and market shifts. Used by strategists and executives to detect score-shifting events before they hit mainstream analysis. Supports investment timing, partnership evaluation, and strategic positioning decisions.
```

For Section 2 (Entity Model), include the full entity definition, observation definition, analysis definition, and dimension definition with Signal Watch's actual values (WatchlistSubject, ClassifiedSignal, Briefing, Competitive Position Score).

For Section 3 (Zone Governance), include the full Green/Yellow/Red zone specifications from the Signal Watch example blocks in the template.

For Section 4 (Cognitive Routing), include the tier mapping with Signal Watch's specific intents mapped to tiers.

For Sections 5-9 (Enrichment), include the Signal Watch examples: wireframe views, voice presets (Strategic Analyst, Executive Brief, Operator Log), seed data (5 AI competitors), knowledge layer, and theme settings.

Section 10 (Build Plan) is auto-generated — include a note that says "Auto-generated by the Foundry compiler from your requirements above."

**For the blank template:** Strip all Signal Watch content but keep:
- All section headers with [REQUIRED]/[ENRICHMENT] markers
- All tables with empty "Your Domain" columns
- All `[PASTE YOUR ... HERE]` placeholders
- The structural guidance text (what an entity is, what an observation is, etc.)

Export:
```typescript
export const FOUNDRY_TEMPLATES: FoundryTemplate[] = [signalWatchTemplate, blankTemplate]
export function getSignalWatchTemplate(): FoundryTemplate { return FOUNDRY_TEMPLATES[0] }
export function getBlankTemplate(): FoundryTemplate { return FOUNDRY_TEMPLATES[1] }
```

## Troubleshooting

**If template text is too long for the API:**
The full template is ~3,000 tokens of user input. Combined with the system prompt (~500 tokens), this is well within context limits for any Tier 3 model. If you hit token limits, the issue is elsewhere.

**If textarea doesn't scroll:**
Verify the className change from `h-40` to `min-h-[10rem] max-h-[50vh] overflow-y-auto`. The `overflow-y-auto` is critical.

**If template recognition doesn't activate:**
The block only activates when the model sees section markers like `## Section 1:`. Verify the template text includes these markers. If the model ignores them, the prompt block wording may need adjustment — but do NOT change the other pipeline blocks.

**If existing hero prompts break:**
The only FoundryPane change that could affect hero prompts is the textarea height. The `injectRandomHeroPrompt` function is unchanged. If hero prompts break, revert only the textarea className change and investigate.
